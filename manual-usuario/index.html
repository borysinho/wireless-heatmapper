Manual de uso de Wireless HeatMapper

Esta guía describe lo que el usuario puede hacer en la aplicación web, la aplicación móvil Android y el portal del cliente. Los nombres de botones, pestañas, campos y mensajes corresponden a la implementación actual.

Usar aplicación web Usar aplicación móvil Ver portal cliente
Panel web Operación RF Usuarios, clientes, proyectos, IA y publicación.
App móvil Trabajo en campo Proyectos, planos, captura WiFi y heatmaps.
Portal cliente Entrega por enlace Datos relevados, modelo IA y mapas interactivos.
Uso Con conexión Las pantallas cargan y guardan información en línea.

Flujo principal

De campo a cliente

1. Preparar
Crear usuarios, clientes y proyectos RF.
2. Relevar
Importar planos y capturar puntos WiFi en campo.
3. Revisar
Previsualizar datos de campo y generar propuestas IA.
4. Entregar
Seleccionar contenido y generar un enlace para el cliente.

Acceso al sistema

Roles principales

Cada rol ve solamente las pantallas necesarias para su trabajo. El administrador trabaja desde la web, el técnico desde Android y el cliente ingresa mediante un enlace compartido.

A

Administrador web

Ingresa a “Wireless HeatMapper”, gestiona “Usuarios”, “Clientes” y “Proyectos RF”, revisa “Datos de campo”, genera “Escenarios IA” y prepara la “Publicación”.

Ver guía web
T

Técnico de campo

Ingresa con correo y contraseña, trabaja en “Mis Proyectos”, importa planos, calibra distancias, captura WiFi y genera mapas desde “Heatmap”.

Ver guía móvil
C

Cliente por enlace

Abre el portal ejecutivo compartido por Bulldog Tech. y revisa solamente el contenido publicado para su proyecto.

Ver guía cliente

Aplicación web

Panel administrativo y operación RF

El panel muestra la navegación “HeatMapper Admin” con los grupos “Centro RF”, “Operación RF” y “Administración”. Desde ahí se accede a “Inicio”, “Proyectos RF”, “Usuarios” y “Clientes”.

01

Iniciar sesión

En la pantalla “Wireless HeatMapper” escriba “Correo electrónico” y “Contraseña”. Use “Iniciar sesión” para entrar al panel.

02

Revisar Inicio

“Inicio” muestra “Wireless HeatMapper”, indicadores como “Proyectos activos”, “Puntos WiFi” y accesos a “Proyectos RF”, “Técnicos y administradores” y “Clientes”.

03

Gestionar Usuarios

En “Usuarios” use “Nuevo técnico”. La tabla muestra “Nombre”, “Correo”, “Rol”, “Estado”, “Creado” y “Acciones” con “Editar”, “Activar” o “Desactivar”.

04

Gestionar Clientes

En “Clientes” use “Nuevo cliente”. Registre “Nombre del cliente” y “Correo de referencia”. La tabla permite “Editar”, “Activar” y “Desactivar”.

05

Gestionar Proyectos RF

“Proyectos RF para publicación” permite “Nuevo proyecto”, filtrar por “Todos los técnicos” y usar “Datos de campo”, “Escenarios IA”, “Publicación”, “Editar”, “Reasignar” y “Eliminar”.

06

Cerrar sesión

Use el botón “Cerrar sesión” del panel lateral cuando termine el trabajo administrativo.

Pantallas web implementadas

Pantalla Qué se ve Qué puede hacer
Usuarios Listado de técnicos y administradores. Crear “Nuevo técnico”, editar datos y cambiar estado.
Clientes Nombre, correo de referencia, estado y fecha de creación. Crear, editar, activar o desactivar clientes.
Proyectos RF Tarjetas de proyectos con técnico, cliente y datos relevados. Crear, editar, reasignar, eliminar y entrar al espacio RF.
Espacio RF del proyecto Pestañas “Datos de campo”, “Escenarios IA” y “Publicación”. Revisar datos reales, generar propuestas y preparar enlaces.

Aplicación móvil Android

Trabajo del técnico en campo

La aplicación móvil muestra las pantallas “Mis Proyectos”, “Planos”, editor de plano, “Captura WiFi” y “Heatmap”. Úsela con conexión disponible durante el relevamiento.

01

Ingresar

Complete “Correo electrónico” y “Contraseña”. Si la conexión falla, la pantalla muestra “Reintentar”.

02

Mis Proyectos

Busque con “Buscar por proyecto o cliente…”. Use “Nuevo proyecto” para crear uno, o el menú “Opciones” para “Archivar” o “Eliminar”.

03

Nuevo proyecto

Registre “Nombre del proyecto *”, seleccione “Cliente” o “Sin cliente”, agregue “Descripción (opcional)” y presione “Crear proyecto” o “Guardar cambios”.

04

Planos

Use “Importar plano”. En “Identificar plano” complete “Nombre del área” y “Descripción opcional”. Si no hay planos, aparece “Importar primer plano”.

05

Preparar el plano

En el editor puede “Medir distancia”, escribir “Metros entre los dos puntos”, “Calibrar”, definir “Área”, “Generar área con IA” y “Eliminar área”.

06

Captura WiFi

Entre con “Captura”. La pantalla “Captura WiFi” permite “Modo puntual”, “Modo continuo — 15 s”, “Modo continuo — 30 s” y “Modo continuo — 60 s”.

07

Ver puntos

Al capturar, la pantalla muestra estados como “Escaneando y enviando…”. Desde el detalle de un punto puede usar “Eliminar punto”.

08

Heatmap

Use “Crear conjunto”, “Conjuntos”, “Cambiar APs”, “Generar heatmap”, “Información del heatmap”, “Recomendaciones” y “Regenerar heatmap”.

Antes de capturar: verifique que el plano corresponde al área visitada, que la escala ya fue calibrada y que el área de interés tiene al menos tres vértices.

Datos y mapas

Revisión RF en la web

Dentro del “Espacio RF del proyecto” se trabaja con las pestañas implementadas: “Datos de campo”, “Escenarios IA” y “Publicación”. Estas pantallas preparan lo que después verá el cliente.

Datos de campo

La pestaña “Datos de campo relevados” lista registros creados desde la app móvil. Muestra “Registro”, “Plano”, “Vista”, “Origen”, “Banda” y “APs”.

Previsualizar

Use “Previsualizar” para abrir el registro, revisar sus APs y consultar los “Mapas de calor generados”.

Escenarios IA

“Escenarios generados por IA” permite elegir “Plano”, “Datos de campo fuente” y “Parámetros IA”.

Generar propuestas

Complete “Umbral objetivo”, “Resolución”, “APs propuestos” y “Cantidad de propuestas”. Luego use “Generar propuestas”.

Propuestas IA del plano

La tabla muestra “Contenido”, “Plano”, “Vista”, “Origen”, “Banda” y “APs”. También permite “Eliminar propuestas IA”.

Publicación

En “Publicación al cliente” seleccione contenido, defina “Vigencia”, elija “Cliente” y use “Generar enlace” o “Enviar enlace”.

Portal cliente

Consulta de resultados publicados

El cliente abre el enlace compartido y ve el proyecto, el nombre del cliente, un resumen ejecutivo y las áreas publicadas por Bulldog Tech.

Resumen ejecutivo

El portal muestra indicadores “Planos”, “Datos de campo”, “Propuestas IA” y “Heatmaps”.

Áreas de análisis

El cliente alterna entre “Datos relevados” y “Modelo IA”, según el contenido incluido por el administrador.

Generación

En “Generación” el cliente elige contenido, modo “Todos”, “Selección” o “Un AP”, y presiona “Generar heatmap”.

Capas del mapa

En el visor puede activar u ocultar “Heatmap”, “APs” y “Puntos”, además de usar acercar y alejar.

Cuando el enlace no está disponible: el portal muestra “Enlace no disponible” y explica que la publicación no existe, expiró o fue revocada por Bulldog Tech.

Solución de problemas

Mensajes comunes y acciones

Estos mensajes existen en la aplicación actual. Úselos como guía para saber qué revisar antes de repetir la operación.

“El polígono requiere al menos 3 vértices.” Editor de plano y Captura WiFi. Marque al menos tres puntos antes de guardar el área.
“No hay datos de campo relevados para este proyecto.” Pestaña “Datos de campo”. Cree el registro desde “Heatmap” en la app móvil.
“Seleccione al menos un contenido para el cliente.” “Publicación al cliente”. Marque uno o más contenidos antes de generar el enlace.
“Todavía no hay enlaces de cliente para este proyecto.” “Enlaces generados”. Genere un nuevo enlace desde la misma pantalla.
“Enlace no disponible” Portal cliente. Solicite a Bulldog Tech. un enlace vigente y activo.
